# 🧠 Códigos Malucos — Laboratório de Refatoração

*"Código ruim não é erro. Errado é não entender por que ele é ruim."*

---

## ⚠️ Sobre este repositório

Este repositório **não é bonito**.  
**Não é polido**.  
**Não é exemplo de "código limpo"** (ainda).

Aqui eu guardo **códigos intencionalmente confusos, bagunçados e mal escritos** — e suas versões **refatoradas, comentadas e compreensíveis**.

> **Observação:** Códigos gerados com IA | Para estudo profissional
> Feito para Refatorar Codigo e tentar entender Logica

---

## 🎯 Objetivo

| Objetivo | Por que |
|----------|---------|
| **Aprender a ler código bagunçado** | No mercado, você vai pegar sistema legado sem documentação |
| **Treinar refatoração** | Transformar código confuso em algo limpo e compreensível |
| **Documentar o processo** | Comentar linha a linha, passo a passo |
| **Não esquecer o que aprendeu** | Cada código refatorado vira consulta rápida |

---

## 🧠 Filosofia

> *"Código ruim não é erro. Errado é não entender por que ele é ruim."*

Aqui, a bagunça é **ferramenta de aprendizado**, não vergonha.  
Cada código maluco é um **desafio**: entender, documentar, refatorar.

---

## 📌 Como usar (se você quiser treinar também)

1. Abra um arquivo da pasta `legado/`
2. Tente entender o que ele faz (sem executar, se possível)
3. Documente com comentários linha a linha
4. Refatore (renomeie variáveis, extraia métodos, simplifique)
5. Salve a versão limpa em `refatorado/`
6. Compare as duas versões e veja a evolução

---

## ⚠️ Aviso importante

- Esses códigos **não são exemplos de boas práticas** (por isso estão em `legado/`)
- Eles foram criados **deliberadamente confusos** para treinamento
- As versões refatoradas representam **um caminho possível**, não o único
- Você pode refatorar de forma diferente! Isso é até melhor para aprender

## 🎯 O que treinar com cada código

| Código | Tópicos para refatorar |
